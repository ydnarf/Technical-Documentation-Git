Basic Configuration

Configure Name that appear in commits

git config --global user.name "example"

Configure Email

git config --global user.email "example@gmail.com"

Colored frame for commands

git config --global color.ui true
Starting repository

Start GIT in the project folder

git init

Clone the repository from github or bitbucket

git clone url

Add all files for the commit

git add .

We do the first commit

git commit -m "Text identifying why the commit was made".

Upload to the repository

git push origin master

GIT ADD

Add all files for the commit

git add .

Add the file for commit

git add archivo

We add all the files for the commit, omitting the new ones.

git add --all

Add all files with the specified extension

git add *.txt

We add all files within a directory and a specific file extension

git add docs/*.txt

Add all files into a directory

git add docs/
GIT COMMIT

Load the changes made in the HEAD

git commit -m "Text identifying why the commit was made".

Add and upload changes to the HEAD

git commit -a -m "Text that identifies why the commit was made".

If there are conflicts, it shows them

git commit -a

Add to the last commit, this is not shown as a new commit in the logs. You can specify a new message

git commit --amend -m "Text that identifies why the commit was made".
GIT PUSH

Upload to the repository

git push origien branch

We upload a tag

git push --tags
GIT LOG

Displays commit logs

git log

Shows the changes in the commits

git log --oneline --stat

Show graphs of commits

git log --oneline --graph
GIT DIFF

Displays changes made to a file

git diff
git diff --staged
GIT HEAD

Pulls a file from the commit

git reset HEAD file

Returns the last commit that was made and staging changes.

git reset --soft HEAD^

Returns the last commit and all changes.

git reset --hard HEAD^

Returns the last 2 commits and all changes.

git reset --hard HEAD^^

Rollback merge/commit

git log
git reset --hard commit_sha
GIT REMOTE

Add remote repository

git remote add origin url

Change remote

git remote set-url origin url

Remove repository

git remote rm name/origin

Show list of repositories

git remote -v

Displays remote branches

git remote show origin

Clear all deleted branches

git remote prune origin
GIT BRANCH

Create a branch

git branch nameBranch

List the branches

git branch

Command -d deletes the branch and joins it to the master

git branch -d nameBranch

Eliminate without asking

git branch -D nameBranch
GIT TAG

Displays a list of all tags

git tag

Create a new tag

git tag -a verison - m "this is version x".
GIT REBASE

Rebasing is used when working with branches, this makes the branches catch up with the master without affecting the master

It joins the current branch with the master, this cannot be seen as a merge.

git rebase

When a conflict occurs you do not give the following options:

when we resolve conflicts --continue to continue the rebasing sequence where it was paused

git rebase --continue

Omit the conflict and go on your way

git rebase --skip

Returns everything to the beginning of the overrun

git reabse --abort

To make a rebasing to a specific branch

git rebase nameBranch
OTHER COMMANDS

  • Lists a current status of the repository with list of modified or added files

    git status

  • Removes a file from the HEAD and sets it to not-worked status

    git checkout -- file

  • Create a branch based on an online one

    git checkout -b newlocalbranchname origin/branch-name

  • Checks for new changes and updates the repository

    git pull origin nameBranch

  • Cambiar de branch

    git checkout nameBranch/tagname

  • Join the current branch with the specified branch

    git merge nameBranch

  • Verifies changes in the online repository with the local repository

    git fetch

  • Delete a file from the repository

    git rm archivo
Fork

Download remote from a fork

git remote add upstream url

Merge with master of a fork

git fetch upstream
git merge upstream/master
